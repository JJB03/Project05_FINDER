# 6. 자체 평가 의견
## 6-1. 개별 평가
- 홍준범
  - 설계 단계부터 배포까지 모든 과정을 경험하면서 설계의 중요성을 절실히 깨달았습니다.
화면 설계를 열심히 했다고 생각했지만, 실제 개발 과정에서는 
계획과 다르게 흘러가는 부분이 많았습니다.

또한, 
ERD를 작성하는 요령과 실력이 부족하여 
여러 개의 조인을 필요로 하는 테이블이 생겼습니다.
ERD를 더 잘 설계했더라면 부족한 시간 안에 
더 많은 일을 할 수 있었을 것이라고 생각합니다.

그래도 한 달 동안 밤을 새우며 비동기와 동기그리고 MVC패턴 기반의 아키텍처 기본기를 확실하게 다질 수 있었습니다. 
이를 통해 어떻게 해야 테이블을 효율적으로 설계할 수 있는지 알게 되었습니다.
기능적으로는 아직 많이 미흡하지만, 
다음 프로젝트에 대해 자신감을 가질 만큼 성장한 것 같아 기쁩니다.

- 유상준
  - 기능 정의서와 화면 설계를 충분히 작성했다고 생각했지만, 
실제로 기능을 구현하면서 부족한 부분이 드러났습니다. 
이를 통해 기능 정의서와 화면 설계를 작성할 때는 충분한 시간을 가지고 
꼼꼼히 작성해야 한다는 교훈을 얻었습니다.

백엔드 개발 측면에서는 비동기 요청을 보내는 부분에서 초기에는 어려움이 있었으나, 
프로젝트를 진행하면서 부족한 부분을 찾아보며 실력을 향상시킬 수 있었습니다. 
어려운 부분이 있으면 겁먹고 미루는 경향이 있었지만, 
프로젝트를 통해 자신감을 얻고 어려운 부분도 도전할 수 있는 생각을 가지게 되었습니다.

이번 프로젝트를 통해 기능 정의서와 화면 설계를 철저히 하는 것의 중요성을 깨달았고, 
어려운 기술적 문제도 도전할 수 있는 자신감을 가지게 되었습니다.

- 정주빈
 -비동기를 안 쓰고 이력서에 모든 데이터를 한 번에 집어넣으려고 하니 
골머리를 앓아 어떤 식으로 설계를 해야 수월하게 작업을 진행할 수 있는지,
작업 순서에 대한 이해가 필요하는 것을 알게 되었습니다.

Java 코드를 짜는데 아직 어려워서 이점을 보안하기 위해 
많은 코드들을 작성해야 한다는 것을 느꼈습니다.
sql에서 외래키를 안 쓰고 다른 방법을 사용해서 
연결할 수 있다는 것을 뒤늦게 알게 되었으며 
다음 프로젝트 할 때에는 기본키로만 쓰려고 합니다.

비동기 사용 방법에 대해 깨닫게 되었으며 
팀프로젝트에서 개인의 몫을 다 하려면 아직 노력이 많이 필요하다는 것과 
서로 협동하면서 하는 것이 얼마나 중요한지를 알게 되었습니다.

   
- 백승헌
  - 쉽지만은 않았습니다. 기획부터 시작해서 요구사항과 기능을 정의하는 것,
ERD와 테이블 및 데이터베이스 정의까지 개발이라는 일이 
그저 코드를 작성하는 일이 아니라 그 이상의 것임을 여실히 깨닫게 되었습니다. 

협업은 각자가 한 곳을 바라보며 가야하고, 같은 생각을 공유하는 것이 
가장 중요한 일이라는 점을 명심하게 되었습니다. 
깃허브를 활용하는 방법도 익힐 수 있었습니다.

기본적인 CRUD 기능과 비동기 요청, 자바 스프링 부트와 타임리프, 
html,css,js 를 이용한 뷰를 작성하는 작업까지 개인적으로도 많은 성취가 있었습니다.

미약하지만 성취감을 느낄 수 있었던  좋은 경험이었습니다.
앞으로 계속해서 걸어갈 길에 초석이 될 수 있을 것 같습니다.

  

## 6-2. 종합 평가
